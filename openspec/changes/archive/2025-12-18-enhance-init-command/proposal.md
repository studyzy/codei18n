# 提案：增强 init 命令

## 为什么
目前的 `codei18n init` 命令功能非常基础，仅生成一个默认的配置文件。用户在初始化后，通常还需要手动执行 `scan`、`map update` 和 `translate` 等一系列操作才能真正开始使用工具。此外，当前的配置初始化逻辑没有充分利用可能存在的全局配置（用户级配置），导致用户需要在每个项目中重复配置 API Key 等敏感信息。

## 目标
增强 `init` 命令，使其成为一个“一键就绪”的初始化入口。具体目标包括：
1. **智能配置继承**：初始化项目配置时，优先继承全局配置（`~/.codei18n/config.json`），并自动剥离敏感信息（如 API Key）以防意外提交。
2. **自动构建映射**：初始化时自动扫描当前项目代码，建立初步的注释映射关系。
3. **条件式翻译**：提供选项让用户在初始化时直接触发首次翻译，快速生成本地语言视图。
4. **Git 集成**：自动检测 Git 仓库环境，将配置目录添加到 `.gitignore` 并安装 Git Hooks。
5. **交互式引导**：优化 CLI 交互体验，引导用户完成初始化流程。

## 变更内容
- 修改 `cmd/codei18n/init.go`，引入更复杂的初始化逻辑。
- 复用 `map update` 和 `translate` 的核心逻辑（需确保这些逻辑可被模块化调用）。
- 增加 Git 环境检测和 `.gitignore` 修改逻辑。
- 复用 `hook install` 逻辑。
- 增加新的 CLI 标志（如 `--with-translation`）。

## 影响范围
- CLI 用户接口（`init` 子命令）。
- 配置管理逻辑。
